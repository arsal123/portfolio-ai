RAG Document Query System

An RAG (Retrieval-Augmented Generation) agent application that allows you to upload PDF documents and query all your uploaded data using a simple search box.

  • On the backend an agent takes the query, pull the relevant data and use a model to generate answer. It also list source documents it has used.
  • Features PDF ingestion with chunking (for large pdfs), vector embeddings and AI-powered context-aware answers.
  • Asynchronous processing with observability (Inngest) and workflow orchestration.
  • A web user interface is written in modern Angular 20 (zoneless).
Tech Stack:
  • Python
  • FastAPI async API framework
  • Inngest for orchestration and observability
  • LlamaIndex for document chunking
  • Qdrant vector database
  • Angular 20 UI client (TypeScript)
  • Streamlit another UI client

Incremental Cache Loader

A Java-based application designed to optimize cache loading for Gemfire by implementing incremental cache updates. This is for a highly transactional microservice where transactions were reaching upto 3000 TPS. As the dataset grew, full cache loads became increasingly time-consuming, necessitating a solution that loads only changed data based on version numbers.

  • Independent Java application that loads only incremental data changes to Gemfire cache instead of full data loads.
  • Version-based data synchronization ensuring only outdated cache entries are updated with latest database versions.
  • Polling mechanism using Java threads to continuously monitor database changes and trigger cache updates.
  • Multithreaded architecture to improve application performance and handle concurrent cache update operations efficiently.
  • Served as primary developer for design and implementation, significantly reducing cache load times as data volume increased.
Tech Stack:
  • Java
  • Gemfire distributed cache
  • Java Threading / Multithreading
  • Database polling mechanisms
  • Version-based synchronization
